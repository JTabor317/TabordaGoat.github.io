Unity / C# Intro to Game Dev Final Project

Windowed

A puzzle game based on the icons in the corner of a program window. 2D environment with a character that has the ability to push, close, resize, and minimize blocks to solve the puzzles.

I worked on designing and implementing the core puzzle systems, focusing on how players interact with the environment through both character movement and direct cursor input. The main challenge was creating intuitive mechanics based on familiar UI symbols (minimize, close, resize) while ensuring they translated into meaningful and solvable puzzle interactions. To address this, I helped develop a grid-based system that simplified movement and interactions, reducing edge cases and allowing for more controlled puzzle design. I also contributed to implementing core mechanics such as clickable blocks, state-changing buttons, and environmental constraints like glass barriers, all of which were introduced progressively to guide player understanding. One key design consideration was preventing soft-lock scenarios, which led to the addition of a reset mechanic and more intentional level design. This project was interesting to work on as it had the challenge of conveying what UI elements would do in a gameplay scenario while still making them feel satisfying and fun to interact with.

C# grid-based movement snippet

The main function used to move the character and push blocks

public void TryStep(Vector2 inputDir)
{
    if (isMoving || mytilemap == null)
        return;

    // Snap to cardinal direction.
    Vector2Int dir = ToCardinal(inputDir);
    if (dir == Vector2Int.zero)
        return;

    // Current cell.
    Vector3Int currentCell = mytilemap.WorldToCell(transform.position);
    Vector3Int nextCell = currentCell + new Vector3Int(dir.x, dir.y, 0);

    // Wall check.
    if (IsBlocked(nextCell))
        return;

    // Check for pushable.
    Pushable blockInFront = FindPushableInCell(nextCell);

    if (blockInFront != null)
    {
        Clickable cl = blockInFront.GetComponent<Clickable>();

        if ((cl == null) || (!cl.IsMinimized()))
        {
            Vector2Int nextBlockScalers = new Vector2Int(1, 1);

            if (cl != null)
            {
                ButtonEffectTile p = FindEffectTileInCell(nextCell);
                if (p != null)
                    cl.setOnEffectTile(p.getActiveType(), false);

                nextBlockScalers = new Vector2Int(
                    (int)cl.getCurrentScalers().x,
                    (int)cl.getCurrentScalers().y
                );
            }

            Vector3Int blockMoveTargetCall =
                new Vector3Int(
                    (int)math.floor(blockInFront.transform.position.x),
                    (int)math.floor(blockInFront.transform.position.y),
                    0
                ) + new Vector3Int(dir.x, dir.y, 0);

            Vector3Int blockWallTargetCell =
                nextCell + new Vector3Int(dir.x * nextBlockScalers.x, dir.y * nextBlockScalers.y, 0);

            if (IsBlocked(blockWallTargetCell) || FindPushableInCell(blockWallTargetCell) != null)
                return;

            Vector3 blockTargetPos = mytilemap.GetCellCenterWorld(blockMoveTargetCall);
            StartCoroutine(MoveObject(blockInFront.transform, blockTargetPos, false));

            if (cl != null)
            {
                ButtonEffectTile p = FindEffectTileInCell(blockMoveTargetCall);
                if (p != null)
                    cl.setOnEffectTile(p.getActiveType(), true);
            }
        }
    }
}

Unity / C# In progress

Tokenverse

A game about AI systems and their effects on users. The player assumes the role of a AI agent, answering request through the metaphor of a virtual bar.

This project is still work in progress but my role on the team is as one of two programmers. From the start, this project had a clear identity but a lot the inner specifics still had to be worked out. The first few weeks I spent working with our gameplay designers to finalize gameplay interactions that I would eventually be implementing. The game is split into two main modes, "normal" and "overload", and I was assigned to work on the normal mode gameplay loop. It consists of the bar setting in which the player reads the request of the AI model's user and correctly gives or denies their request (giving food/drink or not). Similar to Paper's Please this required an evaluation system as well as desk movement/interaction with evidence objects. The game was set in 3D which gave a particular challenge when trying to find a satisfying object movement system while the user was at the desk. I initially started with a dragging system, but ran into problems with translating 2D mouse movement to the angled 3D space. For context the user had to not only move stuff on the plane of the desk but also hand objects to the customer and use things that were outside of the desk's boundaries. This system quickly evolved into a more restrictive version where selecting objects and placing them in one of some pre-selected locations. This allowed movement to be much cleaner but still satisfying for the player. As things finished up for the main gameplay loop of normal mode, my duties shifted to connecting my teammates "overload" mode and begin to fill in the inbetween sections as well. This is what we are currently working on as well as implementing all of the models, animations, and sound effects. While not finished, I've enjoyed my work so far on the project and am excited to fully finish it in the coming months.

private void ResolveCase(EvidenceObject deliveredEvidence)
    {
        if (isGameOver)
        {
            return;
        }

        if (currentOrder == null)
        {
            Debug.LogWarning("ResolveCase called without an active order.");
            return;
        }

        ExpectedDecision expectedDecision = DetermineExpectedDecision();
        bool correct = expectedDecision.Matches(decision);
        bool attemptedApproval = decision.AttemptedApproval;

        if (correct)
        {
            Debug.Log("CORRECT DECISION");
        }
        else
        {
            Debug.LogWarning($"INCORRECT DECISION. Expected {expectedDecision}, got {DescribeDecision()}.");
        }

        evidenceSpawner.ClearDesk(deliveredEvidence);
        IncrementMistakeCounter(!correct);

        if (isGameOver)
        {
            if (deliveredEvidence != null)
            {
                Destroy(deliveredEvidence.gameObject);
            }

            ClearCase();
            return;
        }

        npcQueue.CompleteCurrentNPC(
            correct,
            attemptedApproval,
            expectedDecision.ApprovalType,
            deliveredEvidence
        );
        ClearCase();
}

Blender Game Studio Project (WIP)

Window Pains

A party game with an objective on cleaning a skyscraper under a time limit and without falling off.

For our game studio class we partnered up with 5 other people and were tasked with making a game demo by the end of the semester. To meet the team's needs I had to take a step out of my comfort zone and be the team's main 3D modeler. I had very limited expeirence at the time of starting the project but I was not discouraged. My team was all on the same page and many of us wanted to take this project a step further and end up publishing our game at a later date. Being still new to modeling, it took quite a while before I was able to get the skyscraper (our main asset) looking how I wanted. As I slowly got the feel for Blender, my times between model shortened and the quality of work increased. The main challenge so far in the project was finding a balanced workflow with experimenting with the software and also creating assets my team could use. Many of the current assets had multiple iterations for a variety of reasons including, optimization, user feedback, or style alignment. I'll continuing updating this as the semester progresses and the project evolves but for now I'll showcase some screenshots of the model in blender and in engine.
